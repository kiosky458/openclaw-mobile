# Agent 回應修復方案

**問題**: `openclaw agent` 指令執行超過 60 秒無回應  
**測試時間**: 2026-03-09 10:54

---

## 🔍 問題診斷

### 已驗證的事實
1. ✅ HTTPPollManager 正常運作
2. ✅ Server 端接收訊息成功
3. ✅ `openclaw agent` 指令正常執行（無錯誤）
4. ❌ **Agent 執行超過 60 秒無輸出**

### 日誌證據
```
2026-03-09 10:54:10 [INFO] 🚀 執行指令：/home/kiosky/.nvm/versions/node/v22.22.0/bin/openclaw agent --agent main --session-id mobile_test_device_1773024850_1773024850 --message Hi
...
2026-03-09 10:55:10 [ERROR] ⏱️ Agent 執行超時（60 秒）
```

### 可能原因
1. **Claude Sonnet 4.5 API 調用慢** - 可能 10-30 秒
2. **大量工作空間上下文** - MEMORY.md 21KB（超過 20KB 限制）
3. **Gateway 處理延遲** - 多層處理
4. **模型選擇** - Sonnet 4.5 較慢

---

## 🛠️ 解決方案（4 個選項）

### 選項 A: 增加超時時間 ⭐ **推薦**
**複雜度**: 簡單（1 行代碼）  
**效果**: 高（允許 Agent 完成回應）

**修改**:
```python
stdout, stderr = process.communicate(timeout=120)  # 60 → 120 秒
```

**優點**:
- ✅ 最簡單
- ✅ 保留完整功能
- ✅ 適用於所有模型

**缺點**:
- ⚠️ 用戶需要等待更久（最多 120 秒）

---

### 選項 B: 使用更快的本地模型
**複雜度**: 中等（修改模型配置）  
**效果**: 高（1-5 秒回應）

**修改**:
```python
cmd = [OPENCLAW_BIN, 'agent', '--agent', 'main', '--session-id', session_id, 
       '--message', message, '--model', 'qwen2.5:1.5b']
```

**優點**:
- ✅ 回應極快（1-5 秒）
- ✅ 無 API 費用
- ✅ 不依賴網路

**缺點**:
- ⚠️ 回應品質較低（本地小模型）
- ⚠️ 需要 Ollama 運行

---

### 選項 C: 非阻塞 + 後台推送
**複雜度**: 高（重新設計架構）  
**效果**: 極高（即時反饋）

**架構**:
```
Server 端立即返回 → Agent 背景執行 → 完成後推送訊息
```

**優點**:
- ✅ 用戶無需等待
- ✅ 完全非阻塞
- ✅ 可顯示「處理中」狀態

**缺點**:
- ❌ 需要重新設計輪詢機制
- ❌ 需要額外的「完成通知」邏輯
- ❌ 開發時間長

---

### 選項 D: 簡化工作空間上下文
**複雜度**: 低（縮減 MEMORY.md）  
**效果**: 中（可能節省 5-10 秒）

**修改**:
```bash
# 縮減 MEMORY.md 到 20KB 以下
truncate MEMORY.md 到精簡版本
```

**優點**:
- ✅ 減少上下文加載時間
- ✅ 降低 Token 成本

**缺點**:
- ⚠️ 丟失部分記憶
- ⚠️ 節省時間有限（可能只有 5-10 秒）

---
